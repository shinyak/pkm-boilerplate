---

直近 7 日間のリポジトリ活動を振り返る。

## 1. 情報収集（並列）

以下を並列で収集する：

1. **git log 要約**
   `git log --since="7 days ago" --pretty=format:"%h %ad %s" --date=short` を実行し、コミットを機能別に分類する。

2. **Inbox 未処理**
   `00_Inbox/` 内で作成から 3 日以上経過して未移動のファイルを列挙。

3. **停滞プロジェクト**
   `20_Projects/` 内で 14 日以上更新のないディレクトリを列挙（最終更新日を併記）。

4. **孤立ノート**
   `30_Tech_Notes/` 内で他ノートから一切参照されていないファイルを検出（grep によるパス検索で簡易判定でよい）。

## 2. レビュー記録の作成

`10_Journal/YYYY/MM/YYYY-MM-DD_weekly-review.md` を作成し、以下のフォーマットで保存する：

```markdown
# Weekly Review: YYYY-MM-DD

## 活動サマリ（直近 7 日）
- <git log を機能単位で要約>

## 要対応
- [ ] Inbox 未処理: <ファイル名>（<作成日>）
- [ ] 停滞プロジェクト: <プロジェクト名>（最終更新: <日付>）
- [ ] 孤立ノート: <ファイル名>

## 気づき
- <活動の偏り・重複している可能性のあるノート・新しい関心領域など>

## 次週の焦点
- <ユーザーへの問いかけで埋める>
```

## 3. 対話的な仕分け

レビューファイル作成後、`要対応` 項目を 1 件ずつユーザーに提示し、以下のいずれかを選んでもらう：

- **処理**: 適切なディレクトリへ移動する
- **アーカイブ**: `99_Archives/` へ移動する
- **保留**: 次週に持ち越す（理由を記録する）

「次週の焦点」はユーザーの回答を踏まえて最後に埋める。
